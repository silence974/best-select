## 1. 全书策略总纲（必须先统一）

### 1.1 方法论总纲

1. **量价优先**：量是因、价是果；量价结构是最先行信息。
2. **型态判势**：通过底部/头部形态确认多空转折，而非依赖滞后指标。
3. **顺势波段**：主做波段，不做主观猜顶猜底。
4. **风控先于收益**：严格停损 + 资金控管是长期生存前提。
5. **大赚小赔框架**：小亏损（停损）换取大波段（满足幅度）。

### 1.2 交易纪律总纲

- 多头：看支撑，支撑破就撤。
- 空头：看压力，压力站回就撤。
- 单笔初始停损建议：**5%~7%**。
- 决策优先级：
  1) 结构是否成立；
  2) 突破/跌破是否有效；
  3) 量能是否配合；
  4) 风险收益比是否达标。

### 1.3 周期协同总纲

- **主趋势周期**：日线 / 周线（尽量覆盖半年以上）。
- **先机周期**：小时线 / 5 分钟线（用于“长周期临界点”的提前确认）。
- 短周期不是为了短炒，而是为了优化长周期转折入场点。

## 3. 12 招的统一工程化定义（把“书面经验”转成“策略规则”）

### 3.1 统一信号结构（抽象层）

每个形态统一抽象为：

- `pattern_type`：W/M/旗形/头肩/三角形/假突破等
- `direction`：long / short
- `key_level`：颈线或关键支撑/压力
- `pattern_height`：形态垂直高度
- `volume_state`：放量/缩量/背离
- `entry_trigger`：突破/跌破/回踩确认/反抽失败
- `invalid_trigger`：站回（空）或跌回（多）关键位
- `target_1/2`：第一波、第二波满足位

### 3.2 统一入场与失效

- **多头入场**：突破颈线/压力，且收盘确认；理想情形伴随放量。
- **空头入场**：跌破颈线/支撑，且收盘确认；或假突破后跌回关键位。
- **多头失效**：跌回并有效跌破关键颈线/支撑。
- **空头失效**：站回并有效突破关键颈线/压力。

### 3.3 统一停损框架

- 结构停损：关键位失守立即离场。
- 百分比停损：默认 5%~7%。
- 允许二次加码：突破整理上缘/跌破整理下缘时可加码，但必须同步抬升止损纪律。

### 3.4 统一止盈框架（满足幅度）

- 多头：`Target1 = Breakout + Height`，`Target2 = Target1 + Height`
- 空头：`Target1 = Breakdown - Height`，`Target2 = Target1 - Height`

> 本书多处示例都遵循“等幅测量”思想：先确认高低点落差，再从突破/跌破点投射目标位。

## 4. 各策略形态提炼（逐招）

## 4.1 W 底（双底）——多头

**形态要点**
- 双底 + 颈线。
- 颈线突破后，W 底成立。

**触发**
- 突破颈线买入。

**失效/停损**
- 跌回颈线下方视为失效；配合 5%~7% 停损。

**目标**
- 颈线到底部高度做等幅上推。
- 强势时可做第二波目标（再加一段等幅）。

**书中示例思路**
- 中轴约 19、底约 14，Height≈5；突破约 18 后，Target1≈23，Target2≈28。

## 4.2 破底翻——多头

**形态要点**
- 股价在筑底过程中短暂破底（跌破整理下缘）后，快速站回颈线。
- 常被定义为低位“甩轿/洗浮额”结构。

**触发**
- 破底翻后站回关键线可试单。
- 若再突破整理上缘可做确认加码。

**失效/停损**
- 再度跌回并有效跌破下缘颈线则离场。

**目标**
- 先看整理区突破后的波段满足，再结合后续结构延伸。

## 4.3 破底翻（W 底）——多头

**形态要点**
- “破底翻”与“W 底”复合，属底部强化信号。

**触发**
- 站回关键位 + 后续颈线突破。

**失效/停损**
- 重新跌破底部关键线。

**目标**
- 按 W 底等幅与阶段延伸测算。

## 4.4 下飘旗形（波段涨幅满足）——多头

**形态要点**
- 上涨主趋势中的下倾整理（旗形回档）。
- 整理末端若放量上破，多头续攻概率提高。

**触发**
- 突破旗形上缘/关键颈线。

**失效/停损**
- 跌回旗形内或跌破关键支撑。

**目标**
- 以前一主升段幅度/旗形高度做等幅测算。

## 4.5 头肩底——多头

**形态要点**
- 左肩-头-右肩 + 颈线。
- 颈线突破确认底部反转。

**触发**
- 突破颈线介入。

**失效/停损**
- 跌回颈线下方或右肩低点失守。

**目标**
- 头部最低点到颈线高度上推。

## 4.6 收敛三角形底部突破——多头

**形态要点**
- 振幅收敛，低位整理。
- 末端选择向上突破。

**触发**
- 三角形上沿突破 + 量能配合。

**失效/停损**
- 跌回三角内并破关键支撑。

**目标**
- 三角形最大高度上推；强势做第二段。

## 4.7 上飘旗形（波段跌幅满足）——空头

**形态要点**
- 下跌主趋势中的上倾反弹整理（上飘旗）。
- 反弹衰竭后跌破，进入下一段下跌。

**触发**
- 跌破旗形下缘/颈线。

**失效/停损**
- 站回关键压力位或跌破后的高点被突破。

**目标**
- 以前段跌幅做等幅下推。

## 4.8 M 头（双头）——空头

**形态要点**
- W 底的反向结构。
- 跌破颈线后 M 头成立。

**触发**
- 跌破颈线做空。

**失效/停损**
- 站回颈线之上；配合 5%~7% 停损。

**目标**
- 头部到颈线高度下推。
- 可计算第二波跌幅满足。

## 4.9 假突破——空头

**形态要点**
- 突破后无法站稳，迅速跌回关键位以下。
- 常伴随高位量价背离或“带量不涨”。

**触发**
- 跌回颈线/整理上缘以下确认。
- 可在翻黑破线时加码。

**失效/停损**
- 再次站回并有效突破假突破高点。

**目标**
- 按头部区间或前段波动做跌幅满足。

## 4.10 头肩顶——空头

**形态要点**
- 左肩-头-右肩形成头部。
- 颈线跌破确认反转。

**触发**
- 跌破颈线做空。

**失效/停损**
- 站回颈线或突破右肩反压。

**目标**
- 头部到颈线高度下推。

## 4.11 头肩顶假突破——空头

**形态要点**
- 头肩顶环境下向上假突破失败。

**触发**
- 跌回关键位下方时做空。

**失效/停损**
- 站回头部关键压力区。

**目标**
- 先看头肩顶跌幅满足，再看延伸段。

## 4.12 收敛三角形头部跌破——空头

**形态要点**
- 高位收敛后选择向下突破。

**触发**
- 跌破三角下沿 + 量能放大。

**失效/停损**
- 站回三角并突破下破 K 高点。

**目标**
- 三角最大高度下推。

## 7. 可直接开工的策略开发规格（从“提炼”到“实现”）

## 7.1 模块拆分

1. **数据层**：多周期 OHLCV 同步（日/周/60m/5m）。
2. **形态层**：W/M/头肩/旗形/三角形/假突破识别器。
3. **确认层**：突破有效性（收盘、实体、量能、回踩）。
4. **交易层**：入场、加减仓、止损、分批止盈。
5. **评估层**：胜率、盈亏比、回撤、期望值、分形态表现。

## 7.2 核心接口建议

- `detect_patterns(df, tf) -> List[PatternSignal]`
- `validate_breakout(signal, rules) -> ValidatedSignal`
- `build_trade(signal, account, risk_rules) -> TradePlan`
- `manage_position(position, bar, manage_rules) -> Action`

## 7.3 参数建议

- `stop_loss_pct`: 0.05~0.07
- `risk_per_trade`: 0.005~0.015（净值风险）
- `break_confirm_bars`: 1~3
- `volume_ratio`: 1.2~2.0（按品种调参）
- `target_projection`: `pattern_height`
- `allow_second_wave`: true/false

## 8. 与原书一致的执行原则（避免“会看不会做”）

1. **等待结构完成再交易**（等待是一种纪律）。
2. **进场即带停损**（没有停损就不算策略）。
3. **满足位分批兑现**（不把盈利单变回亏损单）。
4. **不迷信单一指标**（量价+结构才是核心）。
5. **持续复盘历史样本**（案例库是胜率提升关键）。

## 9. 一句话结论（全量版）

《多空转折一手抓》的完整策略体系可还原为：

- 以**量价结构**为先导，
- 以**12 类形态信号**为触发，
- 以**突破/跌破有效性**为确认，
- 以**5%~7% 严格停损 + 等幅满足止盈**为风控骨架，

这套体系天然适合工程化：规则清晰、可参数化、可分模块回测、可跨市场复用。

## 10. 场外基金（支付宝）应用补充：持仓变动与实时估值如何纳入信号

> 目的：在场外基金“日频净值确认 + 盘中信息不完整”的约束下，把盘中可获得信息用于**预警**，而非替代正式信号。

### 10.1 三层信号架构（推荐）

1. **结算确认层（主信号）**
   - 以收盘后可确认的数据（基金净值/跟踪指数收盘）判断 `entry_trigger` 与 `invalid_trigger`。
   - 该层决定正式建仓、减仓、清仓。
2. **盘中代理层（辅助预警）**
   - 用“基金重仓股当日变动 + 指数/ETF实时走势 + 实时估值”估算当日净值方向。
   - 仅用于提前准备动作（如降低风险敞口上限、预设次日调仓）。
3. **执行风控层（约束层）**
   - 盘中代理信号不得越权替代结算确认层。
   - 若盘中和收盘冲突，以收盘确认信号为准。

### 10.2 重仓股变动推算（Nowcast）

- 设基金披露持仓权重为 `w_i`，重仓股盘中收益为 `r_i(t)`，估计基金当日收益：

  `r_hat_fund(t) = Σ(w_i * r_i(t)) + epsilon`

- 其中 `epsilon` 表示：
  - 持仓披露滞后导致的偏差；
  - 未披露持仓、现金仓位、衍生品头寸影响；
  - 当期调仓导致的风格漂移。
- 实操建议：叠加指数收益回归修正，提升稳健性。

### 10.3 实时基金估值的使用原则

- 可用，但应**降权**：
  - 作为辅助确认项，不作为独立触发项；
  - 在高波动时段（开盘/尾盘）仅做方向参考。
- 原因：实时估值通常来自估算模型，噪声和偏差在行情剧烈时会放大。

### 10.4 置信度分层与动作映射

- **High**：重仓股推算、指数代理、实时估值三者同向。
  - 动作：允许 10%~20% 试探仓或提前降 10%~20% 风险仓。
- **Mid**：三者中两者同向。
  - 动作：仅记录预警，不执行主仓变动。
- **Low**：信号分歧。
  - 动作：忽略盘中提示，等待收盘确认。

### 10.5 与原策略框架的衔接

- 盘中代理层新增字段建议：
  - `nowcast_return`
  - `nowcast_confidence`（high/mid/low）
  - `nowcast_source_agreement`（0~3）
- 交易计划层新增字段建议：
  - `pre_action`（none/test_buy/test_reduce）
  - `pre_action_cap`（如 0.1/0.2）
  - `final_action_after_close`（最终以收盘确认）
- 不改动主逻辑：`entry_trigger` / `invalid_trigger` / `target_1/2` 仍是正式决策核心。

### 10.6 风险提示（必须落地）

1. 持仓披露时滞会造成系统性估计误差。
2. 基金经理调仓快时，历史重仓参考价值下降。
3. 实时估值在极端行情中可能显著偏离。
4. 场外申赎确认时滞会放大“信号正确但成交偏离”的执行风险。

### 10.7 场外基金执行铁律

- 盘中信息用于“准备”，收盘确认用于“执行”。
- 预警层只做小比例动作，主仓位必须等待正式信号。
- 所有动作必须可回测、可审计、可复盘。
